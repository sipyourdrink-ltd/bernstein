# N75 — Enterprise Support Tiers

**Priority:** P3
**Scope:** small (10-20 min)
**Wave:** 3 — Enterprise Readiness

## Problem
There is no defined support structure — enterprise buyers cannot evaluate response time guarantees or choose a support plan that matches their operational requirements.

## Solution
- Create `SUPPORT.md` defining three support tiers:
  - **Community:** GitHub Issues, best-effort response, no SLA
  - **Professional ($499/mo):** 24-hour SLA, dedicated Slack channel, priority issue triage
  - **Enterprise (custom pricing):** 4-hour SLA, dedicated support engineer, custom integrations, quarterly business reviews
- Content is structured for use on a publishable pricing page
- Include escalation paths and contact methods for each tier

- [ ] `SUPPORT.md` exists with three clearly defined support tiers
- [ ] Community tier: GitHub Issues, no SLA

- [ ] Enterprise tier: custom pricing, 4h SLA, dedicated engineer
