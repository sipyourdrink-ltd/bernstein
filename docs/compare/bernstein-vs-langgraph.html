Bernstein vs LangGraph

CLI agent orchestration vs LangChain's graph-based LLM node scheduling.

LangGraph is LangChain's graph-based orchestration framework for building stateful multi-agent workflows where LLM nodes are connected by conditional edges and state transitions. Bernstein orchestrates real CLI coding agents (Claude Code, Codex, Gemini CLI) with deterministic Python scheduling, git worktree isolation, and external test-based verification. LangGraph is powerful but complex. Bernstein is narrower in scope but simpler to use for its target domain: software development.

What each tool is

LangGraph is a framework for building stateful, multi-actor applications with LLMs. Built on top of LangChain, it models workflows as directed graphs where nodes are functions (often LLM calls) and edges are conditional transitions. It supports checkpointing, human-in-the-loop interrupts, and streaming. Workflows are Python code that defines graph structure, state schema, and edge conditions. LangGraph is general-purpose -- it can orchestrate any LLM-based workflow, not just coding.

Bernstein is a task-based orchestrator specifically for CLI coding agents. The interface is bernstein -g "your goal". Internally, it decomposes the goal, creates tasks, assigns each to a CLI agent process, runs them in isolated git worktrees, and verifies results with a deterministic janitor (pytest + ruff). No graph definition required. No LangChain dependency.

Architecture comparison

LangGraph defines workflows as state machines. Nodes are functions. Edges are conditional transitions.

Python code defining StateGraph
    |
    v
Graph compiler
    |
    +-- Node: "planner" (LLM call)
    |       |
    |       +-- edge: if needs_research -> "researcher"
    |       +-- edge: if ready -> "coder"
    |
    +-- Node: "researcher" (LLM + tools)
    |       |
    |       +-- edge: -> "planner" (loop back)
    |
    +-- Node: "coder" (LLM + code execution)
    |       |
    |       +-- edge: if tests_fail -> "debugger"
    |       +-- edge: if tests_pass -> END

State passed between nodes. Checkpointed per step.

Bernstein uses a flat task queue with deterministic dispatch.

bernstein -g "goal"
    |
    v
Planner (1 LLM call) -> task list
    |
    v
Task server (deterministic Python)
    |
    +-- Task A -> claude  (isolated worktree) -> janitor -> merge
    +-- Task B -> codex   (isolated worktree) -> janitor -> merge
    +-- Task C -> gemini  (isolated worktree) -> janitor -> merge

No graph definition. No edge conditions. No state schema.

Mental model complexity

LangGraph requires you to understand: state schemas (TypedDict or Pydantic), node functions, conditional edge functions, graph compilation, checkpoint serialization, and the LangChain ecosystem (chains, tools, callbacks, runnables). The learning curve is significant. Building a production LangGraph workflow often takes days of iteration.

Bernstein's mental model is: goal decomposes into tasks, tasks are assigned to agents, agents are verified by the janitor. The entire user interface is one command: bernstein -g "your goal". For power users, YAML plan files offer explicit control. No graph theory required.

This simplicity is by design. For software development orchestration, the task queue pattern (decompose, dispatch, verify, merge) is sufficient. Graph-based orchestration adds flexibility that coding workflows rarely need.

Ecosystem dependency

LangGraph is part of the LangChain ecosystem. In practice, using LangGraph effectively means also using LangChain (for chains, tools, and prompts), LangSmith (for tracing and monitoring), and often LangServe (for deployment). This is a rich ecosystem but also a significant dependency. If LangChain's abstractions do not match your mental model, you fight the framework.

Bernstein has zero ecosystem dependencies. It is a standalone Python package that wraps installed CLI tools. No chains, no runnables, no callbacks. The orchestrator is ~800 lines of Python you can read in an afternoon.

Cost comparison

LangGraph's graph-driven execution may route through multiple LLM nodes before producing output. A planner-researcher-coder graph with retry loops can generate 10-20 LLM calls per task. LangSmith tracing adds a per-event cost for monitoring.

Bernstein's scheduling is zero-token. The only LLM spend is on actual coding. Built-in cost tracking is free -- no separate monitoring product required.

When to use LangGraph

  • You are building a custom stateful agent application. Complex conversation flows, tool-use chains, human-in-the-loop workflows -- LangGraph's graph model handles these well.
  • You need fine-grained control over execution flow. Conditional edges, retry loops, parallel branches with join conditions.
  • You are already in the LangChain ecosystem. If you use LangChain chains, tools, and LangSmith, LangGraph is the natural orchestration layer.
  • You need token-level streaming. LangGraph streams output token-by-token through the graph.
  • Your agents are LLM nodes, not CLI tools. If your workflow is LLM-call-to-LLM-call with tool use, LangGraph's node model is a good fit.

When to use Bernstein

  • You want to orchestrate real CLI coding agents. Claude Code, Codex, Gemini CLI -- installed tools, not graph nodes.
  • You want the simplest possible interface. bernstein -g "your goal". No graph definition, no state schema, no edge conditions.
  • You do not want LangChain ecosystem lock-in. Bernstein is standalone. Zero framework dependencies.
  • You need git-native isolation. Each agent works in its own worktree.
  • You need built-in verification. Janitor runs tests and linter. No custom verification node to build.
  • You want deterministic scheduling. Auditable, reproducible, zero-token coordination.
  • You want built-in cost tracking. No LangSmith subscription required.

Frequently asked questions

What is the difference between LangGraph and Bernstein?

LangGraph is LangChain's graph-based orchestration for stateful multi-agent workflows with LLM nodes. Bernstein orchestrates real CLI coding agents with deterministic scheduling, git worktree isolation, and external test verification.

Do I need LangChain to use Bernstein?

No. Bernstein has zero dependency on LangChain. It wraps installed CLI coding agents as subprocess adapters.

Is LangGraph or Bernstein simpler to learn?

Bernstein is simpler. The entire interface is: bernstein -g 'your goal'. LangGraph requires understanding state graphs, node functions, edge conditions, checkpoint serialization, and the LangChain ecosystem.
