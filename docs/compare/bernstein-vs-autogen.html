Bernstein vs AutoGen

File-based CLI orchestration vs Microsoft's in-memory multi-agent conversation framework.

AutoGen is Microsoft's multi-agent conversation framework where Python agents communicate through in-memory message passing with configurable conversation patterns. Bernstein uses file-based state in .sdd/, wraps 18+ installed CLI coding agents as subprocess adapters, applies deterministic Python scheduling with zero coordination tokens, and isolates each agent in its own git worktree. AutoGen is a general-purpose agent framework. Bernstein is purpose-built for software development orchestration.

What each tool is

AutoGen (by Microsoft Research) is a multi-agent conversation framework. You define agents as Python objects with LLM backends, tool access, and conversation policies. Agents communicate by passing messages in configurable patterns: two-agent chat, group chat with a manager, sequential chains. State lives in memory during execution. It is designed for building multi-agent applications across domains -- research, analysis, code generation, customer support.

Bernstein is a task-based orchestrator for CLI coding agents. You provide a goal. Bernstein decomposes it, creates task files in .sdd/backlog/, spawns CLI agents (Claude Code, Codex, Gemini CLI) in isolated git worktrees, and verifies results with a deterministic janitor (tests + linter). All state is persistent plain files. The orchestrator is ~800 lines of deterministic Python -- no LLM calls for scheduling.

Architecture comparison

AutoGen uses configurable conversation patterns. Agents exchange messages in memory.

AutoGen runtime (in-memory)

    |       |
    |       v (conversation)
    +-- UserProxyAgent (code execution, human proxy)
    |       |
    |       v (conversation)
    +-- GroupChatManager (manages multi-agent turns)

Bernstein uses task files and isolated CLI processes.

    |
    v
Task server (FastAPI, file-based state in .sdd/)
    |
    +-- Task A -> claude  (isolated worktree) -> janitor -> merge
    +-- Task B -> codex   (isolated worktree) -> janitor -> merge

State: .sdd/ files. Survives crashes. Git-committed.

State: files vs memory

AutoGen's in-memory state means a process crash loses all conversation history and progress. If an agent is halfway through a complex task and the Python process dies, you start over.

Bernstein's file-based state (.sdd/) means task progress, agent assignments, and completion records survive crashes. If the process restarts, the task server reads the backlog from disk and resumes. Tasks that were in-progress get requeued. Nothing is lost.

This matters for long-running sessions. A 4-hour overnight run with 20 tasks is common in Bernstein. If the machine reboots at hour 3, Bernstein picks up where it left off. AutoGen would need to re-execute everything.

Agent model: SDK vs CLI

AutoGen agents are Python objects. You define their capabilities, tools, and conversation behavior in code. This gives you unlimited flexibility but ties you to Python and requires SDK integration for every capability.

Bernstein agents are installed CLI tools. Claude Code, Codex, Gemini CLI, Aider, Cursor -- whatever is on your PATH. The adapter layer is a thin subprocess wrapper. When any CLI tool vendor ships an improvement, Bernstein uses it automatically. No SDK update required.

This also means Bernstein supports agents that do not have Python SDKs. Cursor, Kiro, Roo Code, Goose -- these are desktop or CLI tools that AutoGen cannot easily wrap. Bernstein supports all of them through the generic adapter.

Cost comparison

AutoGen's conversation-based coordination means agents exchange messages to make decisions. A group chat with 3 agents and 4 rounds generates 12+ LLM calls before reaching a conclusion. For coding tasks with clear decomposition, this overhead adds cost without value.

Bernstein's deterministic scheduling costs zero tokens. The only LLM spend is on the actual coding work.

When to use AutoGen

  • You are building a custom multi-agent application. Research pipelines, data analysis, customer support -- AutoGen's conversation patterns give you fine-grained control.
  • You need agents to converse. If agents need to challenge, refine, and iterate on each other's reasoning, AutoGen's group chat is designed for this.
  • You want maximum flexibility. AutoGen's Python API lets you define any agent behavior, any tool, any conversation pattern.
  • You are in the Microsoft ecosystem. AutoGen integrates with Azure OpenAI, Microsoft Semantic Kernel, and other Microsoft tooling.
  • The task is not software development. If your agents are not writing code to files, git isolation and janitor verification are irrelevant.

When to use Bernstein

  • You want to orchestrate CLI coding agents. Claude Code, Codex, Gemini CLI, Aider -- installed tools, not SDK abstractions.
  • You need crash-resilient state. File-based state in .sdd/ survives process crashes. In-memory state does not.
  • You need git isolation. Each agent works in its own worktree. No merge conflicts during execution.
  • You need built-in verification. Janitor runs tests and linter before merge. No agent self-assessment.
  • You want deterministic scheduling. Zero tokens on coordination. Auditable, reproducible task assignment.
  • You want headless overnight operation. bernstein --headless --budget 20 runs until the queue is empty.

Frequently asked questions

What is the difference between AutoGen and Bernstein?

AutoGen is Microsoft's Python-based multi-agent conversation framework with in-memory state. Bernstein is a CLI agent orchestrator with file-based state, 18+ CLI agent adapters, deterministic scheduling, and git worktree isolation.

Does Bernstein support more AI agents than AutoGen?

Bernstein supports 18+ CLI agents via thin subprocess adapters. AutoGen agents are Python classes -- you define behavior in code rather than wrapping installed tools.

Is AutoGen or Bernstein better for software development tasks?

Bernstein is purpose-built for software development: git worktree isolation, janitor verification, model routing, and headless overnight operation. AutoGen is a general-purpose agent framework better suited for research, analysis, and custom applications.
