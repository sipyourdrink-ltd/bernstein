Bernstein vs Parallel Code

Automated orchestration vs manual multi-terminal parallelism for AI coding agents.

Parallel Code refers to running multiple coding agent sessions simultaneously in separate terminals or tmux panes with human coordination. Bernstein automates the entire parallel workflow: it decomposes goals into tasks, assigns each to the cheapest capable model, runs agents in isolated git worktrees, verifies output with a deterministic janitor, and merges results without conflicts. The question is whether you want to be the orchestrator or have software do it.

What each tool is

Parallel Code refers to the pattern of running multiple independent coding agent sessions simultaneously -- typically via tmux, multiple terminal windows, or a session manager. The parallelism is human-coordinated: you decide what tasks to run, assign them to sessions, and monitor the output.

Bernstein automates that coordination. You provide a goal in natural language. Bernstein decomposes it into subtasks, assigns each to an agent process with the appropriate model and role, runs them in isolated git worktrees, verifies results with a janitor (tests + linter), and merges the work.

Architecture comparison

Parallel Code: you are the scheduler, the verifier, and the merge coordinator.

You (the human orchestrator)
    |
    +-- Terminal 1: claude --prompt "add /users endpoint"  -> you review
    +-- Terminal 2: claude --prompt "write tests for auth"  -> you review
    +-- Terminal 3: codex  --prompt "update API docs"       -> you review
         |
         v
    You merge the output, resolve conflicts, check tests

Bernstein: the scheduler, verifier, and merge coordinator are all software.

bernstein -g "add user management: endpoints + tests + docs"
    |
    v
Task planner (LLM) -> 3 tasks: [endpoints, tests, docs]
    |
    +-- Task A -> claude  (worktree: feat/task-a) -> janitor -> merge
    +-- Task B -> codex   (worktree: feat/task-b) -> janitor -> merge
    +-- Task C -> gemini  (worktree: feat/task-c) -> janitor -> merge

You return when it is done. Or set --headless and do not return.

The coordination tax

Managing parallel agent sessions manually costs real attention: writing prompts for each session, monitoring multiple outputs, running tests after completion, merging without conflicts, and tracking costs. For 2-3 sessions on a short task, this tax is acceptable. For 5+ sessions on a complex feature, it consumes most of the time savings from parallelism.

When manual parallel is better

  • You have 2-3 short tasks you have done before. The overhead of setting up Bernstein is not worth it.
  • You want to watch the agent work. Parallel terminal sessions let you see reasoning in real time.
  • You need fine-grained control over each session. Intervening mid-task, redirecting the agent.
  • The tasks are not independent. Sequential dependencies mean parallelism does not help.

When Bernstein is better

  • You have 4+ tasks. Human coordination does not scale linearly; Bernstein does.
  • You want verified output. Janitor runs tests after every task.
  • You want to run overnight or in CI. bernstein --headless --budget 15.00 runs unattended.
  • You are doing this repeatedly. Setup investment pays off after a few runs.
  • You want cost data. Per-task, per-model cost logging.
  • You want model routing. Cheap models for simple tasks, expensive models for complex ones.

Migration path

Parallel Code users typically adopt Bernstein after experiencing: two sessions modifying the same file creating merge conflicts, a session "completing" but tests failing, a session silently erroring out hours ago, or spending more time coordinating sessions than the tasks themselves took.

Frequently asked questions

What is the difference between Parallel Code and Bernstein?

Parallel Code is the pattern of running multiple coding agent sessions in separate terminal windows with human coordination. Bernstein automates the entire process: task planning, model routing, git worktree isolation, janitor verification, and merge coordination.

When should I switch from manual parallel coding agents to Bernstein?

Switch when you experience merge conflicts from overlapping sessions, discover test failures after "completed" sessions, find silent errors in unmonitored sessions, or spend more time coordinating 5+ sessions than the tasks themselves take.

Can Bernstein run coding agents overnight without supervision?

Yes. Run bernstein --headless --budget 15.00 and it operates until the backlog is empty or the budget runs out, retrying failures automatically.
