Bernstein vs CrewAI

Deterministic CLI orchestration vs Python-native LLM-driven agent scheduling.

CrewAI is a popular Python framework for multi-agent orchestration where agents are Python classes with LLM-driven task delegation. Bernstein takes a fundamentally different approach: it wraps any installed CLI coding agent (Claude Code, Codex, Gemini CLI, and 15+ others) with deterministic Python scheduling that spends zero LLM tokens on coordination. CrewAI is better for building custom agent applications in Python. Bernstein is better for orchestrating real CLI coding agents on software development tasks.

What each tool is

CrewAI is a Python-native multi-agent framework. You define agents as Python classes with roles, goals, and backstories. You define tasks with expected outputs. The framework uses LLM calls to decide which agent handles which task, how to delegate, and when to escalate. Agents communicate through a shared memory system. It is designed for building custom agent applications -- customer support, research pipelines, content generation.

Bernstein is an orchestrator for CLI coding agents. You provide a natural language goal. Bernstein decomposes it into tasks using a single LLM planner call, then hands off to deterministic Python code: task assignment based on role/complexity rules, model routing via a cost-aware bandit, git worktree isolation per agent, and janitor verification (tests + linter) before merge. The agents are installed CLI tools (Claude Code, Codex, Gemini CLI), not Python SDK classes.

Scheduling: LLM vs deterministic

CrewAI's LLM-driven scheduling means the framework spends tokens deciding which agent should handle a task, how to break it down, and when to delegate. For complex workflows, this can mean dozens of LLM calls before any actual work is done. The scheduling itself is non-deterministic -- the same crew definition can produce different execution paths on different runs.

Bernstein's scheduling is pure Python code: role-based task assignment, complexity-based model routing, dependency resolution. The scheduling logic is deterministic, auditable, and reproducible. It costs zero tokens. The only LLM call is the initial goal decomposition by the planner -- and even that is skippable if you provide a YAML plan file.

For software development tasks where the decomposition is straightforward (frontend + backend + tests + docs), deterministic scheduling is faster and cheaper. For novel, ambiguous tasks where the right decomposition is itself a research problem, LLM-driven scheduling can explore approaches that static rules would miss.

Agent model: SDK classes vs CLI tools

CrewAI agents are Python classes. You write code to define their role, tools, and behavior. This gives you fine-grained control but requires Python integration for every agent capability.

Bernstein agents are installed CLI tools. Claude Code, Codex, Gemini CLI, Aider, Cursor -- whatever is on your PATH. Each runs as a subprocess with a prompt. No Python SDK integration required. The adapter layer is thin: it launches the CLI, passes the prompt, and reads the output.

This means Bernstein immediately benefits from any improvement the CLI tool vendors ship. When Claude Code gets a new capability, Bernstein uses it automatically. CrewAI requires manual integration of new model capabilities through Python code.

Cost comparison

CrewAI's coordination overhead scales with workflow complexity. A 5-agent crew with delegation may spend 20-30 LLM calls on scheduling before producing output. At $0.01-0.03 per call, that is $0.20-0.90 in coordination overhead alone.

Bernstein's coordination overhead is fixed at zero tokens (or one planner call if using goal-based decomposition). The only LLM spend is on the actual coding work.

When to use CrewAI

  • You are building a custom agent application. Customer support, research pipelines, content generation -- CrewAI's Python SDK gives you fine-grained control over agent behavior.
  • You need agents with custom tools. CrewAI's tool system lets you attach any Python function as an agent capability.
  • You want LLM-driven task delegation. For novel, ambiguous tasks where the right decomposition is not obvious, LLM scheduling can explore creative approaches.
  • You prefer defining agents in code. Python classes with explicit roles, goals, and backstories.
  • Your agents do not need git isolation. If the work is not file-based (API calls, database queries, web research), git worktrees are irrelevant.

When to use Bernstein

  • You want to orchestrate real CLI coding agents. Claude Code, Codex, Gemini CLI, Aider -- installed tools, not SDK abstractions.
  • You want zero coordination overhead. Deterministic scheduling spends no tokens on figuring out which agent does what.
  • You need git-native isolation. Each agent works in its own worktree. No merge conflicts during execution.
  • You need built-in verification. Janitor runs tests and linter before any merge. Agents cannot self-certify completion.
  • You want cost tracking and budget caps. Per-task, per-model cost logging with hard budget limits.
  • You want headless, overnight operation. bernstein --headless --budget 20 runs until complete.

Frequently asked questions

What is the difference between CrewAI and Bernstein?

CrewAI is a Python framework where you define agents as Python classes with LLM-driven task delegation. Bernstein wraps any installed CLI coding agent with deterministic Python scheduling -- zero LLM tokens on coordination.

Does CrewAI or Bernstein cost more to run?

CrewAI spends LLM tokens on agent coordination and task delegation. Bernstein's scheduling is pure Python with zero LLM overhead. For coordination-heavy workflows, Bernstein significantly reduces costs.

Can I use CLI coding agents like Claude Code with CrewAI?

CrewAI agents are Python SDK classes. Bernstein wraps 18+ installed CLI agents as subprocess adapters -- no Python SDK integration required.
