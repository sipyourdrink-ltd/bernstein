Bernstein vs Paperclip

Engineering orchestrator vs AI company simulator for managing coding agents.

Paperclip is an AI company simulator that models agents as employees in an organization with org charts, budgets, and governance hierarchies. Bernstein is an engineering tool that spawns coding agents, runs them in isolated git worktrees, verifies their output against tests and linter, and merges working code. Paperclip answers "how do I organize AI agents like a company." Bernstein answers "how do I get code shipped faster using multiple agents in parallel."

What each tool is

Paperclip (launched 2026-03-04; MIT, Node.js + React) is an AI-company control plane. It models AI agents as employees in an organization: they have roles, report to managers, operate within budgets, follow org-chart hierarchies, and receive scheduled heartbeats. The coordinator is an LLM. It supports Claude Code, Codex, Cursor, and OpenClaw plus bash and HTTP hooks.

Bernstein (Apache 2.0, Python) is a multi-agent orchestrator for CLI coding agents. It breaks a goal into tasks, spawns agents in isolated git worktrees, verifies their output (tests, lint, file checks), and merges the results. The orchestrator is deterministic Python -- zero LLM tokens spent on coordination. It supports 17 CLI agents and runs anywhere Python runs.

Paperclip uses LLM calls for coordination -- managing hierarchies, routing tasks through org structures, heartbeat processing.

The fundamental difference

Paperclip answers: "How do I organize and govern a fleet of AI agents like a company?"

Bernstein answers: "How do I get code shipped faster using multiple agents in parallel?"

These are genuinely different problems. Paperclip cares about organizational structure -- who reports to whom, what budget each team has, how goals cascade through a hierarchy. Bernstein cares about engineering output -- did the tests pass, did the linter pass, can this branch merge cleanly.

When to use Paperclip

  • You are managing AI agents as a business function. Multiple teams, budgets, approval chains, governance requirements.
  • You want a visual dashboard. Non-technical stakeholders need to see what agents are doing.
  • You need multi-company support. Consultancy or MSP deploying agents across client organizations.
  • The problem is coordination, not code. Agents do diverse work (not just coding) and need organizational structure.

When to use Bernstein

  • You want to ship code. Implement features in parallel and merge them into a working branch.
  • You want zero coordination overhead. No LLM tokens on scheduling. Deterministic task assignment and verification.
  • You use diverse CLI agents. 17 adapters. Mix Claude, Codex, Gemini, and Aider in the same session.
  • You want git-native safety. Worktree isolation, conflict detection, janitor verification.
  • You work in terminals. CLI-native, works over SSH, runs in CI.

Frequently asked questions

What is the difference between Paperclip and Bernstein?

Paperclip is an AI company management platform that models agents as employees with org charts, budgets, and governance hierarchies. Bernstein is a coding agent orchestrator that spawns agents, verifies their code output against tests, and merges working branches.

Does Bernstein have a web UI like Paperclip?

Bernstein is CLI-native with a TUI dashboard and a JSON status endpoint. It does not ship a React web UI.

Can Paperclip and Bernstein be used together?

Yes. Paperclip can manage the organizational layer while Bernstein handles the engineering execution within each team.
