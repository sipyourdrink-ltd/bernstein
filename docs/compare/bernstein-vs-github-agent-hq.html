Bernstein vs GitHub Agent HQ

Open-source multi-agent orchestration vs GitHub's proprietary multi-agent coding system.

GitHub Agent HQ is a proprietary multi-agent coding system built into GitHub that runs Claude, Codex, and Copilot simultaneously on tasks within the GitHub ecosystem. Bernstein is the open-source alternative: it runs locally or in CI, supports 18+ CLI agents across any git host, uses deterministic Python scheduling that costs zero tokens on coordination, and gives you full ownership of the orchestrator code under Apache 2.0.

What each tool is

GitHub Agent HQ is a multi-agent coding system built into GitHub. It runs inside GitHub's infrastructure, uses GitHub's AI models, and integrates tightly with pull requests, issues, code review, and GitHub Actions. The orchestrator logic is GitHub's proprietary code.

Bernstein is an open-source orchestrator for CLI coding agents. It runs on your machine (or in CI), wraps whichever CLI tools you have installed, and stores state in plain files. The orchestrator is ~800 lines of deterministic Python with no LLM calls on coordination. You can read, modify, and extend all of it.

Architecture comparison

GitHub Agent HQ runs in GitHub's cloud with native access to your repo, issues, and PR history.

GitHub issue / PR / comment
    |
    v
Agent HQ coordinator (GitHub infrastructure)
    |
    +-- Agent 1 (Claude)   -- GitHub-managed execution
    +-- Agent 2 (Codex)    -- GitHub-managed execution
    +-- Agent 3 (Copilot)  -- GitHub-managed execution
         |
         v
    PR created, CI triggered, code review requested

Bernstein runs on your hardware with isolated git worktrees and external janitor verification.

bernstein -g "goal"  (terminal, CI, SSH)
    |
    v
Task server (local FastAPI, deterministic Python)
    |
    +-- Task A -> claude  (isolated worktree) -> janitor -> merge
    +-- Task B -> codex   (isolated worktree) -> janitor -> merge
    +-- Task C -> gemini  (isolated worktree) -> janitor -> merge

State: .sdd/ files (backlog, runtime, metrics, config)

Cost model

GitHub Agent HQ cost is bundled into Copilot Enterprise ($39/user/month) or charged as additional compute. For teams already on Copilot Enterprise, Agent HQ feels "free" -- it is included in the seat cost.

Bernstein cost = model API tokens only. No seat license, no subscription. A typical medium-complexity task using mixed-model routing costs $0.15-0.20 in API tokens. A 10-task session costs roughly $1.50-2.00.

When Agent HQ is cheaper: If your team already pays for Copilot Enterprise and runs fewer than ~200 Agent HQ tasks per user per month.

When Bernstein is cheaper: If you are not on Copilot Enterprise, if you run high task volumes, or if you route tasks aggressively to Haiku/Gemini.

Where Agent HQ is genuinely better

  • GitHub integration. Agent HQ turning a GitHub issue into a pull request, with CI triggered and reviewers requested, requires zero configuration.
  • No local setup. No installation, no Python environment, no API key wiring.
  • Multi-model coordination within a task. Agent HQ can have Claude review Codex's output within the same task execution.
  • Enterprise support. GitHub sells SLAs, security reviews, and compliance documentation.

When to use Agent HQ

  • Your workflow is entirely GitHub-native. Issue tracking, code review, CI -- all on GitHub.
  • Your team already pays for Copilot Enterprise. Incremental cost is near zero.
  • You want zero local setup. No Python, no API keys, no terminal.
  • You need enterprise SLAs and vendor support.

When to use Bernstein

  • You host code outside GitHub. GitLab, Bitbucket, self-hosted Gitea, or a plain git remote.
  • You want model flexibility. Mix Claude, Codex, Gemini, and Qwen in the same session.
  • You want to own the orchestrator. 800 lines of Python you can read, modify, fork, and audit.
  • You want cost transparency and optimization. Per-task cost logging with bandit routing.
  • You want self-evolution. bernstein --evolve improves prompts, routing, and templates over time.
  • You want headless, overnight, or CI operation. bernstein --headless runs until complete.
  • You are not on GitHub.

Frequently asked questions

Is Bernstein an open-source alternative to GitHub Agent HQ?

Yes. Bernstein provides the same multi-agent coding orchestration pattern -- parallel agents, task decomposition, verification before merge -- but runs locally, supports any git host, works with 18+ CLI agents, and is Apache 2.0 licensed.

Can Bernstein work outside of GitHub?

Yes. Bernstein works with any git repository on any host: GitHub, GitLab, Bitbucket, self-hosted Gitea, or a plain git remote.

Is GitHub Agent HQ or Bernstein cheaper for multi-agent coding?

It depends on your existing subscriptions. If you already pay for Copilot Enterprise ($39/user/month), Agent HQ's incremental cost is near zero. If you are not on Copilot Enterprise, Bernstein costs only model API tokens -- typically $0.15-0.20 per task with mixed-model routing.
