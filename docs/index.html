How it works

Short-lived agents, file-based state

The key insight: agents aren't long-lived workers. They're spawned fresh, do 1-3 tasks, and exit. No idle loops. No context rot. State lives in .sdd/ files between sessions.

1

Describe a goal

Pass a plain-English goal or a bernstein.yaml seed file with pre-defined tasks.

2

Manager decomposes

An Opus manager agent breaks the goal into typed tasks with roles, scopes, and dependencies.

3

Spawner routes and launches

The spawner picks the right model and effort for each task, then launches a fresh CLI agent session.

4

Janitor verifies

Before marking a task done, the janitor checks completion signals: file exists, test passes, API responds.

5

Loop until empty

When all tasks are done the system exits. Or keep it running in evolve mode for continuous improvement.

Features

Everything a multi-agent system needs

No hand-waving. Each feature is a concrete, observable behaviour.

⚡

Parallel agents

Up to 6 agents run simultaneously on independent tasks. Dependency-blocked tasks wait automatically — no manual coordination.

🔄

Self-evolution

Run bernstein --evolve and Bernstein analyses its own metrics, proposes improvements, validates them in a sandbox, and applies low-risk ones automatically.

🗂️

Agent catalogs

Load specialist agents from YAML/Markdown catalogs. Sync from remote marketplaces. Each agent brings its own role prompt and model preferences.

✅

Built-in verification

Tasks define completion signals (path_exists, test_passes, api_responds). The janitor checks them before closing — agents can't self-report done.

🧠

Smart model routing

Opus for architecture and security review. Sonnet for implementation. The router decides based on task scope, complexity, and role — not a flat config.

🔌

Any CLI agent

Claude Code, Codex, Gemini CLI, Qwen-Coder — all supported via thin adapters. The orchestrator is model-agnostic; adapters handle the CLI differences.

Quick start

Up in three commands

# Install (Python 3.12+)
pip install bernstein

# Initialise a project workspace
cd my-project && bernstein init

# Run with a plain-English goal
bernstein -g "Add pagination to the users API endpoint"

Or drop a bernstein.yaml in the project root and run bernstein with no arguments.

Bernstein is open-source and works with the CLI agents you already have.
