Agent-independent orchestration. Ship while you sleep.

Bernstein is an open-source multi-agent orchestration platform that coordinates AI coding agents to work on software projects in parallel. It supports 17 CLI agents including Claude Code, Codex, and Gemini CLI, uses deterministic Python scheduling, and ships with built-in verification, cost tracking, and SOC 2 audit trails.

Think of it as what Kubernetes did for containers, but for AI coding agents. You declare a goal. The control plane decomposes it into tasks. Short-lived agents execute them in isolated worktrees — like pods. A janitor verifies the output before anything lands.

How it works

Short-lived agents, file-based state, deterministic orchestration

Agents aren't long-lived workers. They spawn fresh, do 1-3 tasks, and exit. No idle loops, no context rot. All state lives in .sdd/ files — git-native, inspectable, reproducible. Tasks transition through OPEN → IN_PROGRESS → DONE → CLOSED, with the final CLOSED state applied only after janitor verification. Explore the full architecture and concepts

  1. Describe a goal Pass a plain-English goal or a bernstein.yaml config. Multi-repo workspaces supported.
  2. Manager decomposes An Opus manager breaks the goal into typed tasks with roles, scopes, dependencies, and completion signals.
  3. Router selects models Each task gets the right model and effort level. Opus for architecture, Sonnet for implementation, free-tier for simple fixes. Cost-aware by default.
  4. Agents work in parallel Fresh CLI agents spawn, work in isolated git worktrees, and exit. Dependency-blocked tasks wait automatically.
  5. Janitor verifies, loop continues Before marking done, the janitor checks completion signals: file exists, tests pass, API responds. Failed tasks get retried or escalated.

Features

Everything a multi-agent system needs

No hand-waving. Each feature is a concrete, observable behaviour you can inspect in code.

Orchestration

Parallel agents

Multiple agents run simultaneously on independent tasks in isolated git worktrees. Dependencies resolve automatically — no manual coordination.

Smart model routing

Opus for architecture and security. Sonnet for implementation. Free-tier for formatting. The router picks based on task scope, complexity, and role — not a flat config.

Multi-cell orchestration

Organise agents into cells — self-contained teams with a manager and workers. A VP agent coordinates across cells for large-scale projects.

A/B model testing

Run the same task with different models or configurations and compare results. Automated scoring picks the winner by cost, speed, and output quality.

Batch mode

Queue hundreds of tasks from a YAML file and let Bernstein grind through them unattended. Ideal for large-scale refactors and migration campaigns.

Reliability

Built-in verification

Tasks define completion signals (path_exists, test_passes, api_responds). The janitor verifies them — agents can't self-report done.

File-based state

All state lives in .sdd/ files. Git-native, inspectable, reproducible. No database required for single-instance. State survives crashes and restarts. Tasks follow a strict lifecycle: OPEN → IN_PROGRESS → DONE → CLOSED (verified by the janitor before closing).

Cost budgeting

Set a dollar cap with --budget 5.00. The tracker warns at 80%, alerts at 95%, and hard-stops agents at 100%. No more runaway API bills.

Verified cache and quota-aware status

Only verified response-cache entries can auto-complete work. Provider health and quota snapshots are polled separately and shown in bernstein status and /status.

Token analyzer

Per-task and per-agent token usage breakdowns. Tracks input, output, and cache-read tokens across providers. Surfaces waste patterns and suggests model downgrades.

Agent cache

Caches agent outputs keyed by task hash. Identical re-runs skip spawning entirely. Cache entries are verified before reuse — stale or failed entries are evicted automatically.

Extensibility

Mix models in the same run. No vendor lock-in. Full adapter reference and feature matrix

Agent catalogs

Load specialist agents from YAML/Markdown catalogs. Sync from remote marketplaces. Each agent brings its own role prompt, model preferences, and capabilities.

MCP tool access

Agents can use MCP servers (stdio/SSE) for external tools — GitHub, filesystem, web search. The orchestrator manages MCP server lifecycle automatically.

Plan templates

Ship with built-in plan templates for common workflows: migration, refactor, greenfield feature, test backfill. Extend with your own YAML templates in templates/plans/.

Enterprise & Compliance

HMAC-chained audit log

Every orchestrator decision is logged in a tamper-evident, HMAC-chained JSONL file. Daily rotation, Merkle tree sealing, bernstein audit verify-hmac validates chain integrity. SOC 2 Type II evidence export built in.

Execution WAL with crash recovery

Hash-chained write-ahead log. On crash, startup replays uncommitted WAL entries so no work is silently lost. bernstein verify --determinism proves two runs made identical decisions.

Policy limits & multi-tenant isolation

Fail-open policy limits with ETag caching, per-tenant resource quotas, permission delegation. Role-based access control (admin, operator, viewer) with SSO/SAML/OIDC and OAuth 2.0 PKCE.

Sensitive data protection

PII output gate blocks sensitive data from leaving the system. Differential privacy for metrics. State encryption at rest with key rotation. EU AI Act risk tagging on every task.

CI autofix pipeline

bernstein ci fix <url> parses a failing GitHub Actions run, creates a fix task, and opens a PR. bernstein ci watch monitors continuously and auto-fixes.

Intelligence

Contextual bandit router

Learns the optimal (model, effort) pair for each task type over time. Starts with heuristics, converges to data-driven routing. Saves 15-30% on token costs after the first few runs.

Cross-model verification

Route completed diffs to a different model for independent review before merge. A Sonnet agent writes the code; an Opus judge reviews it. No single-model blind spots.

Knowledge graph

Builds a dependency graph of your codebase — files, functions, imports, test coverage. Tasks are routed to agents with relevant context. bernstein impact-analysis shows what breaks if you change a file.

Cost anomaly detection & forecasting

Z-score-based anomaly detection flags unusual spend patterns in real time. Spend forecasting predicts total run cost based on current trajectory. Hard-stop before surprise bills.

Semantic caching

LLM responses cached by semantic similarity, not exact match. Identical questions with different phrasing hit the cache. Saves tokens on repeated patterns across tasks.

Observability

Prometheus, OTel, Datadog, Grafana

Built-in /metrics endpoint for Prometheus. OTel exporter presets for distributed tracing. Datadog integration. Auto-generated Grafana dashboards. Wire into your existing stack in minutes.

Test impact analysis

Identifies which tests are affected by agent changes. Flaky test detection with statistical tracking. Duplicate code detection across agent outputs. Run only what matters.

Release train orchestration

Coordinate multi-stage releases across teams and repos. Stage gates, rollback plans, deployment sequencing. From feature branch to production with automated verification at each step.

Operations

Run --evolve and Bernstein analyses its own metrics, generates improvements via a visionary/analyst pipeline, validates in a sandbox, and applies safe ones automatically.

Quick start

Up in two commands

# Install (Python 3.12+)
pipx install bernstein   # or: uv tool install bernstein

# Run with a plain-English goal
bernstein -g "Add pagination to the users API endpoint"

# Or run in self-evolution mode
bernstein run --evolve --max-cycles 10

Common questions about AI agent orchestration

Is Bernstein better than CrewAI?

Bernstein and CrewAI solve different problems. Bernstein orchestrates real CLI coding agents (Claude Code, Codex, Gemini CLI) with deterministic scheduling — no LLM tokens spent on coordination. CrewAI uses Python SDK classes with LLM-driven scheduling. Choose Bernstein for production-grade multi-agent coding workflows; choose CrewAI for custom Python agent pipelines. Full comparison

What is the best way to orchestrate AI coding agents?

The most effective approach uses deterministic scheduling (not LLM-based), git worktree isolation for each agent, and automated verification before merging. Bernstein implements all three: agents work in parallel on isolated copies, the janitor verifies their output, and only verified work lands in main. Learn the architecture

Can you run multiple AI agents on the same codebase at once?

Yes. Bernstein spawns each agent in its own git worktree — an isolated copy of your repository. Agents work simultaneously without conflicts. The orchestrator merges verified results back to main. A single run can coordinate 17 different AI coding agents. Get started

How much does AI agent orchestration cost?

Bernstein is free and open-source (Apache 2.0). You pay only for the AI models your agents use. Bernstein includes built-in cost tracking, model routing to minimize spend (cheap models for boilerplate, expensive for architecture), and cost anomaly detection. Run bernstein cost to see spend by model and task.
